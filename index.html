Tutorial | Express.js | MongooseDB
The main things I learned:
  • I gained a better understanding of middleware.
  • I learned how to build authentication with JSON Web Tokens and used cookies. I didn’t use JSON Web Tokens or cookies in my past projects.
  • I learned how to set up emails and send them to users’ email addresses so that they can reset their password if they forgot it.
  • I also learned that when setting up the “contact us” feature, I can set the email to go to the owner of the website, so that when they click reply, it will use the user’s email address.
  • I learned how to upload images with Multer and save them to Cloudinary.
  • I learned how to manage state with Redux Toolkit, although I prefer the implementation I am learning in another tutorial better.
  • I utilized the "express-async-handler" package.
  • I set up error handling in Express.
Main things I learned on my own
  1. I changed the upload image with Multer to use memory/buffer instead of saving the file on the server. A free server didn't want me to use up their space, and it was wasteful anyway since I was just uploading it to Cloudinary.
  2. I also set it up so that you could easily change Multer methods with env.
  3. I didn’t like uploading images to Cloudinary from the frontend by setting it up to not require a signature, as the tutorial showed me. So I looked at the Cloudinary documentation to see how I could use signatures that would be sent from my API to the client to upload the image from there.
  4. Another reason I didn't like how the tutorial did this was that anyone could flood my Cloudinary storage by uploading files again and again. So I found that I could give the file a name, and if I tried to upload a file with the same name, it would overwrite the old file. I liked this because it kept my storage clean. And the signature made sure they couldn't name the file anything they wanted. I used the user ID for profile pictures because they are unique and the SKU for the product for the same reason.
  5. I also added the image to be deleted when a product was deleted. The documentation said nothing about needing to put the holder name as part of the public_ID. I was confused as to why, but I at least figured it out by guessing.
  6. I didn’t like that users could get to the login page when they were already logged in, so I made it redirect them away from that page if they were logged in.
  7. Because the implementation of pagination had changed since the tutorial was made, I practiced understanding the documentation for it so that I could implement it as if it were completely new to me. The way they were showing me how to use it was not the way I would have written it.
  8. Having the Chrome DevTools open can cause a crash.

Building an inventory app, I learned the most about authorizing users using JWT authentication, using tokens with cookies, how middleware works (versus using them without knowing why it worked), uploading files, and setting up emails.

I learned some frontend things, but those were shown to me without being explained.
